📊 Анализ ежедневных продаж — интерактивный дашборд


🎯 Бизнес-задача

Компания (интернет-магазин) хочет видеть динамику продаж по дням, понимать, какие товары приносят основную выручку, и быстро реагировать на изменения. Заказчик — менеджер по продукту.

Ключевые вопросы дашборда:

- Какова общая выручка и количество проданных единиц за выбранный период?
- Какие 5-10 товаров дают максимальную выручку?
- Как меняются продажи день ото дня?

📦 Исходные данные

Датасет: [SQL Practice Dataset 1 (Easy)](https://www.kaggle.com/datasets/nudratabbas/sql-practice-dataset-1-easy-queries/data) — три CSV-таблицы:
- `customers` — информация о покупателях
- `products` — информация о товарах
- `orders` — информация о заказах

🧹 Подготовка и очистка данных (SQL + DBeaver)

При импорте CSV заметил несоответствия:
- `order_date` имел тип `VARCHAR(100)` → изменил на `DATE`
- `price` был `INT` → изменил на `DECIMAL(10,2)` (чтобы избежать потери точности)
- При попытке изменить `order_id` с `VARCHAR` на `INT` получил ошибку.  
Проверив данные, обнаружил, что ID содержит буквенный префикс (например, `P008`). Привык, что когда учил, то везде в ID были только целые числа, ну, на ошибках учатся.
Решение: оставил тип `VARCHAR`, так как первичные ключи не обязаны быть целыми числами — важны уникальность и неизменность.

Ещё я заметил, что в датасете отсутствуют явные первичные и внешние ключи. Таблицы были просто набором данных без связей.

[Это не критично для CSV-файлов, но решил добавить ключи, чтобы](https://github.com/JeyvVegas/sales-dashboard-portfolio/blob/main/Sales-Dashboard-Portfolio/SQL/schema_setup.sql)
- обеспечить целостность данных,
- ускорить JOIN-запросы,
- приблизить структуру к реальным производственным базам.


📊 Разработка дашборда в Power BI

1. Импорт данных и предобработка
- Из DBeaver результат SQL-запроса экспортирован в CSV (UTF-8, разделитель `,`).
- Загрузка в Power BI Desktop через «CSV».

2. Нормализация модели 
Исходная таблица `daily_sales` содержала повторяющиеся названия товаров. Для оптимизации и соответствия лучшим практикам:
- Создана отдельная таблица-справочник `daily_sales_ref`(удалены все столбцы, кроме `product_id` и `product_name` → удаление дубликатов).
- В модели установлена связь «один-ко-многим» между `daily_sales_ref[product_id]` и `daily_sales[product_id]` Для визуализаций используется `product_name` из справочника.

3. Создание таблицы календаря (для временной аналитики)
- Calendar = CALENDAR(MIN('daily_sales'[date]); MAX('daily_sales'[date]))

4. Ключевые меры DAX
Для анализа продаж созданы следующие показатели:

 | Total Revenue | `SUM('daily_sales'[total_revenue])` | Общая выручка за выбранный период |

 | Total Quantity | `SUM('daily_sales'[quantity])` | Общее количество проданных единиц |

 | Revenue Previous Day | `CALCULATE([Total Revenue]; DATEADD('Calendar'[date]; -1; DAY))` | Выручка за предыдущий день (для расчёта динамики) |

 | Day-over-Day Change | `DIVIDE([Total Revenue] - [Revenue Previous Day]; [Revenue Previous Day]; 0)` | Процент изменения выручки день ко дню |

 | Revenue Rolling 7 Days | `CALCULATE([Total Revenue]; DATESINPERIOD('Calendar'[date]; LASTDATE('Calendar'[date]); -7; DAY))` | Скользящая сумма за 7 дней (сглаживает пики) |

 | Количество товаров | `COUNTROWS('daily_sales_ref')` | Сколько уникальных товаров в ассортименте |

5. Визуализации и интерактивность
 | График | `Calendar[date]`, мера `Total Revenue` и `Total Quantity` | Динамику общей выручки по дням. Позволяет увидеть тренды, пики и падения. |

 | Слайсер по дате | `Calendar[date]` | Выбор периода: можно задать произвольный диапазон дат (тип «Между»). |

 | Карточка | Общее количество уникальных товаров в ассортименте (мера `Количество товаров = COUNTROWS('daily_sales_ref')`). | Это информационная метрика, а не фильтр. |

 | Гистограмма (топ-10) | `daily_sales_ref[product_name]`, мера `Total Revenue` (значение) | Показывает 10 товаров с наибольшей выручкой. При клике на любой столбец остальные визуализации пересчитываются по выбранному товару. |

 | Таблица детализации | `daily_sales_ref[product_name]`, меры `Total Revenue`, `Total Quantity` | Для каждого товара отображает выручку и количество проданных единиц. Поддерживает сортировку по любому столбцу. |
